# AI 使用說明

## 原則：AI 輔助整理，不接管防災決策

本作品有兩個不同的 AI 使用位置，兩者都不是核心決策依據：

| 元件 | AI 角色 | 核心演算法仍是什麼 | AI 不可做什麼 |
|---|---|---|---|
| 沉默災區偵測 API | 使用 Ollama 把既有雙隊列與原因整理為人可讀摘要。 | `rule_based_mvp` 沉默風險公式。 | 改變排序、加入／移除村里、宣布災害、發布撤離／封路命令。 |
| 志工派工 API | 選用 Ollama 進行異常檢查。 | deterministic 技能、距離、急迫度、可用性匹配。 | 替代派工演算法、隱藏分數／警示、直接決定實際派遣。 |

## 沉默元件的 AI 使用

可用於：

- 將 `silent_watch_queue` 與 `verified_incident_queue` 的已存在欄位整理為摘要。
- 把風險原因轉成容易閱讀的提示。
- 協助管理者了解資料狀態與需要人工確認之處。

不可用於：

- 把模型文字當成災害真實性判定。
- 將 `pending` 回報自動視為可信事件。
- 改寫規則式分數、風險級別或 queue。
- 執行任何實際控制、通報、撤離、封路或資源強制調度。

Ollama 不可用時，規則式 `command_plan` 仍必須可用。

## 志工元件的 AI 使用

可用於：

- 對 deterministic 分配結果做異常檢查或文字提示。
- 提醒協調者查看顯著技能缺口、距離不合理或高急迫度未指派任務。

不可用於：

- 捏造志工技能、位置或可出勤狀態。
- 抹去 `unassigned_tasks`、`warnings` 或低信心分數。
- 在無人員核准下傳送派遣命令。

## 機器學習實驗層

沉默元件可保留 NN scoring layer 的研究程式與 `silent_risk_nn_score`，但其目前 pseudo-label 並非真實災害 ground truth。因此：

- 不宣稱預測準確率。
- 不讓 NN 分數取代正式 `rule_based_mvp` 排序。
- 在輸出中標示 `model_status`。
- 未來若引入真實標註資料，仍需做資料切分、偏誤檢查、校準、版本控管與人因測試。

## 展示時應如何說

正確：

> AI 僅負責整理既有規則結果與標示異常；系統的風險排序與派工建議可被檢視，最終確認與行動仍由人員負責。

錯誤：

> AI 會預測災害、判定災情、直接派志工、或自動救援。
