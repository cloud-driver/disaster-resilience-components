# 資料來源與接入規劃

本作品目前以樣本資料驗證雙元件流程，正式部署時可接入公開防災資料、村里資料、道路資料、災情通報資料與志工資料。

## 資料來源規劃

| 資料類型 | 用途 | MVP 使用方式 | 正式接入方式 |
|---|---|---|---|
| 村里邊界與行政區資料 | 建立分析單位，將風險分數對應到村里 | 使用樣本村里資料 | 接入政府開放資料或 GIS 圖資 |
| 雨量、水位、淹水或土石流風險資料 | 判斷區域災害風險 | 使用模擬風險欄位 | 接入即時觀測資料或災防開放資料 |
| 災情通報資料 | 判斷某區域是否「通報不足」 | 使用樣本通報數 | 串接通報系統、LINE Bot、表單或 API |
| 道路狀態資料 | 判斷交通可及性與救援困難度 | 使用樣本道路狀態 | 接入道路封閉、橋梁、交通中斷資料 |
| 人口與高齡比例資料 | 判斷弱勢風險與主動巡查需求 | 使用樣本人口特徵 | 接入人口統計或村里層級資料 |
| 志工資料 | 進行任務派遣與技能比對 | 使用 `examples/sample_volunteers.json` | 串接志工平台、地方單位或救災團體資料 |

## MVP 階段資料

目前 repo 內提供：

- `examples/sample_silent_zone_output.json`
- `examples/sample_volunteers.json`
- `examples/sample_dispatch_output.json`

這些資料用於驗證資料流，不代表真實災害結果。

## 正式部署注意事項

1. 資料來源需確認授權與更新頻率。
2. 即時資料需處理延遲、缺漏與格式不一致問題。
3. 志工資料涉及個人資料，正式部署時必須取得授權並進行最小化蒐集。
4. 系統輸出應作為決策輔助，不應直接取代指揮中心或現場人員判斷。
